# 需求文档：A 股申万行业成交与买卖分析

> 版本：**v3.0**（看板 + 历史序列）  
> 状态：Phase 1 ✅ · Phase 2/3 待实现  
> 必盈接入：[BIYING_API.md](./BIYING_API.md)  
> 实现方案：[IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md)

---

## 1. 项目目标

基于 **申万行业分类** 与 **必盈 API**，构建 A 股 **成交额 + 主买/主卖** 的多层级分析能力，并通过 **Web 看板** 可视化展示。

| 层级 | 指标 | 看板入口 |
|------|------|----------|
| 全 A | 成交额、主买、主卖 | 页面 1 |
| 申万板块 | 成交额、占比、主买、主卖 | 页面 2、3 |
| 个股 | 成交额、主买、主卖 | 页面 4（从页面 3 下钻） |
| ETF | 成交额 | 本期看板 **不纳入**（无买卖接口，可 Phase 4 单列） |

## 3. 看板需求（Phase 3，本次新增）

### 3.1 通用规则

#### 3.1.1 数值单位

所有金额类展示（图表轴、表格单元格、Tooltip）须 **按数值大小自动选用单位**：

| 绝对值范围 | 显示单位 | 示例 |
|------------|----------|------|
| ≥ 1 亿 | **亿** | `123.45 亿` |
| ≥ 1 千万且 < 1 亿 | **千万** | `8,765 千万` |
| ≥ 1 万且 < 1 千万 | **万** | `456.7 万` |
| < 1 万 | **元** | `9,876 元` |

- 内部存储与计算一律使用 **元（float）**，仅在展示层格式化。
- 占比类字段使用 **百分比**，保留 2 位小数，如 `12.34%`。
- 图表 Y 轴与 Tooltip 使用同一套格式化函数，避免单位不一致。

#### 3.1.2 交易日口径

- 「近 N 日」= 最近 **N 个交易日**（非自然日）。
- 数据缺失的交易日跳过，图表 X 轴只显示有数据的日期。
- 页面 1 的 30 日买卖图：若历史不足 30 个交易日，展示已有全部数据并在页脚标注实际天数。

#### 3.1.3 导航结构

```text
页面 1  全 A 概览
  └─► 页面 2  板块表格（近 5 日）
        └─► 页面 3  板块图表（近 5 日，可折叠）
              └─► 页面 4  板块内个股图表（近 5 日，可折叠）
```

- 顶部统一导航栏：概览 | 板块表格 | 板块图表。
- 页面 3 每个板块行尾有 **「查看个股」** 按钮 → 跳转页面 4，并带上 `sector_code` 参数。
- 页面 4 面包屑：`概览 > 板块图表 > {板块名} > 个股`。

---

### 3.2 页面 1：全 A 概览

**路由**：`/`

| 区块 | 图表类型 | 时间范围 | 数据系列 |
|------|----------|----------|----------|
| A | 柱状图 | **近 5 个交易日** | 全 A **成交额** `turnover` |
| B | 柱状图 | **近 30 个交易日** | 全 A **主买额** `active_buy` |
| C | 柱状图 | **近 30 个交易日** | 全 A **主卖额** `active_sell` |

**交互**

- X 轴：交易日期（`MM-DD` 或 `YYYY-MM-DD`）。
- Y 轴：自动单位（万 / 千万 / 亿）。
- Hover Tooltip：日期、原始值（元）、格式化值、单位。
- 三图纵向排列，宽度自适应（建议单栏，移动端可滚动）。

**数据来源**

`market_daily` 表（或等价历史文件），字段见 §4.2。

---

### 3.3 页面 2：板块近 5 日交易额表格

**路由**：`/sectors/table`

#### 3.3.1 表格结构

- **行**：申万一级板块（约 31 行 + 可选「合计」行）。
- **列组**（每个交易日一组，共 5 组）：

| 子列 | 说明 |
|------|------|
| 成交额 | 板块绝对值，自动单位 |
| 占 A 股比 | `sector.turnover / market.turnover`，百分比 |

示例列头：

```text
板块名称 | 06-09 成交额 | 06-09 占比 | 06-10 成交额 | 06-10 占比 | … | 06-13 成交额 | 06-13 占比
```

- 默认按 **最新交易日占比从大到小** 排序。
- 表头提供排序切换：
  - 按 **最新日占比 ↓**（默认）
  - 按 **最新日占比 ↑**
  - 按 **最新日成交额 ↓**
  - 按 **板块名称 A-Z**

#### 3.3.2 交互

- 点击板块名称 → 跳转页面 3 并定位到该板块（锚点或 query `?sector=`）。
- 占比单元格可用浅色热力底色（占比越高越深），可选实现。

---

### 3.4 页面 3：板块近 5 日图表

**路由**：`/sectors/charts`

每个 **申万一级板块** 一个可折叠卡片（Accordion）：

```text
▼ 银行                                    [查看个股]
    ├─ 柱状图：近 5 日成交额
    ├─ 柱状图：近 5 日主买额
    └─ 柱状图：近 5 日主卖额
▶ 电子                                    [查看个股]
▶ …
```

**交互**

- 默认 **全部折叠** 或 **仅展开第一个**（实现时二选一，文档默认：全部折叠，用户手动展开）。
- 「查看个股」→ 页面 4 `/sectors/{sector_code}/stocks`。
- 板块列表顺序与页面 2 一致（随排序联动，或通过 URL 参数传递排序方式）。

**数据**

`sector_daily`：`turnover`, `active_buy`, `active_sell`，近 5 交易日。

---

### 3.5 页面 4：板块内个股近 5 日图表

**路由**：`/sectors/{sector_code}/stocks`

- 页头展示板块名称、近 5 日板块汇总（成交额 / 主买 / 主卖，一行 KPI）。
- 每个 **成份股** 一个可折叠卡片，结构与页面 3 相同（三张柱状图）。
- 个股顺序：默认按 **最新交易日成交额从大到小**。

```text
▼ 000001 平安银行
    ├─ 近 5 日成交额
    ├─ 近 5 日主买额
    └─ 近 5 日主卖额
▶ 600036 招商银行
…
```

**数据**

`stock_daily`：按 `sector_code` 过滤，近 5 交易日。

**说明**

- 使用 Phase 1 **主编行业** 归属（一股一板块），与 CSV 口径一致。
- 跨板块重复映射的股票只出现在其主行业下。

## 4. 数据需求（Phase 2，看板前置）

看板依赖 **历史序列**，Phase 1 单日 CSV 不满足需求，须新增历史层。

### 4.1 最少历史深度

| 用途 | 最少交易日数 |
|------|-------------|
| 页面 1 成交额、页面 2–4 | **5** |
| 页面 1 买卖 30 日图 | **30** |

### 4.2 逻辑数据模型

#### market_daily（全 A 日汇总，1 行/日）

| 字段 | 类型 | 说明 |
|------|------|------|
| `trade_date` | date | 主键 |
| `turnover` | float | 全 A 成交额（元） |
| `active_buy` | float | 全 A 主买（元） |
| `active_sell` | float | 全 A 主卖（元） |
| `net_active` | float | 主买 − 主卖 |
| `stock_count` | int | 有数据股票数 |
| `snapshot_time` | datetime | 采集时间 |

#### sector_daily（板块日汇总，~31 行/日）

| 字段 | 类型 | 说明 |
|------|------|------|
| `trade_date` | date | 联合主键 |
| `sector_code` | str | 联合主键 |
| `sector_name` | str | 申万一级名称 |
| `turnover` | float | 板块成交额 |
| `turnover_pct` | float | 占全 A 比（冗余存储便于查询） |
| `active_buy` | float | 板块主买 |
| `active_sell` | float | 板块主卖 |
| `net_active` | float | 板块净主买 |
| `stock_count` | int | 成份股数 |

#### stock_daily（个股日明细，~5200 行/日）

| 字段 | 类型 | 说明 |
|------|------|------|
| `trade_date` | date | 联合主键 |
| `stock_code` | str | 联合主键 |
| `stock_name` | str | |
| `sector_code` | str | 主编行业 |
| `sector_name` | str | |
| `turnover` | float | |
| `active_buy` | float | |
| `active_sell` | float | |
| `net_active` | float | |

### 4.3 采集与回填策略

| 场景 | 策略 |
|------|------|
| **每日增量** | 交易日 21:35 cron 跑 `fetch_by_daily.py`，结果 **UPSERT** 进历史库 |
| **首次回填 5 日** | 从已落库的 5 天 CSV/SQLite 积累；或板块/全市场从个股聚合 |
| **首次回填 30 日买卖** | 逐股调用 `history/transaction?lt=30`（约 5208×1 次），本地聚合为 market/sector；**一次性任务** |
| **成交额 30 日** | 若必盈无全市场历史接口，由 `stock_daily.turnover` 按日 SUM 得到 market/sector |

### 4.4 API 频率与约束

| 操作 | 请求量 | 说明 |
|------|--------|------|
| 每日增量 | ~5208（买卖）+ ~1480（ETF，可选）+ ~261 批（成交） | 与 Phase 1 相同 |
| 30 日一次性回填 | ~5208 | 每只股票 `lt=30`，包年约 2 分钟级 |
| 存储增量 | ~5200 行/日 stock_daily | SQLite 足够 |

## 9. 验收标准（Phase 3）

1. 页面 1 三张柱状图数据与 `market_daily` 一致，单位显示正确。
2. 页面 2 表格 5 列日组 × 2 子列，占比之和约等于 100%（未归类股导致略小于 100% 可接受）。
3. 页面 2 排序切换生效，默认占比降序。
4. 页面 3 每板块 3 图可折叠，「查看个股」跳转正确。
5. 页面 4 仅展示该板块成份股，每个股 3 图可折叠。
6. 历史不足 30 日时，页面 1 买卖图展示已有天数并提示。
