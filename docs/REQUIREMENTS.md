# 需求文档：A 股申万行业成交与买卖分析

> 版本：**v3.3**（看板六页面 + 管理页 + 近 5 日历史）  
> 状态：Phase 1 ✅ · Phase 2/3 待实现  
> 必盈接入：[BIYING_API.md](./BIYING_API.md)  
> 实现方案：[IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md)

---

## 1. 项目目标

基于 **必盈 API**，构建 A 股 **成交额 + 主买/主卖** 与 **ETF 成交额** 的多层级分析能力，并通过 **Web 看板** 展示 **近 5 个交易日** 的趋势与对比。

| 层级 | 指标 | 看板入口 |
|------|------|----------|
| 全 A | 成交额、主买、主卖 | 页面 1 |
| 申万板块 | 成交额、占比、主买、主卖 | 页面 2、3 |
| 个股 | 成交额、主买、主卖 | 页面 4（从页面 3 下钻） |
| ETF | 成交额、占 A 股比 | 页面 5、6 |
| 数据管理 | 定时任务、补数、导出、日历 | **页面 7（管理页）** |

**统一时间窗口**：看板页面 1–6 均为 **近 5 个交易日**（非自然日）。

## 3. 看板通用规则

### 3.1 数值单位

所有金额类展示（图表轴、表格、Tooltip）按数值大小 **自动选用单位**：

| 绝对值范围 | 单位 | 示例 |
|------------|------|------|
| ≥ 1 亿 | **亿** | `123.45 亿` |
| ≥ 1 千万且 < 1 亿 | **千万** | `876.5 千万` |
| ≥ 1 万且 < 1 千万 | **万** | `456.7 万` |
| < 1 万 | **元** | `9,876 元` |

- 存储与 API 返回一律 **元（float）**；格式化仅在前端展示层。
- 占比：**百分比**，保留 2 位小数，如 `12.34%`。
- 同一页面内图表 Y 轴、表格、Tooltip 共用一套格式化函数。

### 3.2 交易日口径

- 「近 5 日」= 数据库中最近 **5 个交易日**（按 `trade_date` 降序取 5 条）。
- 某日无数据则跳过，不插零；不足 5 日时展示已有全部日期，页脚标注「当前 N 个交易日」。
- 必盈买卖数据 **21:30** 更新；默认定时 **21:35**，保证买卖与成交可对齐。

### 3.3 占比口径

| 场景 | 公式 |
|------|------|
| 板块占 A 股（页面 2） | `sector.turnover / market.turnover` |
| ETF 占 A 股（页面 5） | `etf.turnover / market.turnover` |

分母均为 **当日全 A 成交额**（`market_daily.turnover`），非 ETF 市场合计。

### 3.4 导航结构

```text
页面 1  全 A 概览
页面 2  板块表格 ──► 页面 3  板块图表 ──► 页面 4  板块个股
页面 5  ETF 表格
页面 6  ETF 图表
页面 7  管理（定时 / 补数 / 导出 / 数据日历）
```

**顶栏导航（建议）**

```text
[概览] [板块表格] [板块图表] [ETF 表格] [ETF 图表] [管理]
```

- 页面 3 每板块 **「查看个股」** → 页面 4 `/sectors/{sector_code}/stocks`。
- 页面 4 面包屑：`概览 > 板块图表 > {板块名} > 个股`。

### 4.7 页面 7：数据管理

**路由**：`/admin`  
**权限**：仅管理员可访问（Basic Auth 或独立 admin token；与普通看板只读分离）。

管理页集中提供 **采集调度、手动补数、数据导出、数据日历** 四类能力，并统一展示 **任务状态与报错信息**。

#### 4.7.1 定时更新配置

| 项 | 说明 |
|----|------|
| 功能 | 配置 **每天自动更新** 当日（最新可用 `trade_date`）全量数据的执行时刻 |
| 默认时间 | **21:35**（Asia/Shanghai） |
| 执行日类型 | **交易日**（默认）或 **自然日**；管理页可切换 |
| 配置项 | 时、分（24 小时制）；是否启用；执行日类型（`trading_day` / `calendar_day`） |
| 交易日模式 | 仅在 **A 股交易日** 触发；周末与法定休市日跳过 |
| 自然日模式 | **每天**到点触发（含周末/节假日）；适用于测试或强制补跑 |
| 更新范围 | 与 `fetch_by_daily.py` 一致：行业映射缓存、个股成交/买卖、板块汇总、全 A 汇总、ETF 成交，并 UPSERT `history.db` |
| 状态展示 | 最近一次定时任务：开始/结束时间、目标 `trade_date`、状态（成功/失败/运行中）、耗时 |
| 报错展示 | 失败时展示 **错误摘要** + **日志尾部**（可展开完整 log）；支持复制 |

**说明**

- 界面「当日」指任务运行时 API 能提供的 **目标交易日期**（`trade_date`），写入 DB 以该日期为键。
- **21:35** 为产品缺省，与必盈买卖 21:30 更新衔接；管理员可在管理页调整。

#### 4.7.2 手动指定日期更新

| 项 | 说明 |
|----|------|
| 功能 | 选择 **YYYY-MM-DD** 某一交易日，手动触发该日数据采集并写入历史库 |
| 场景 | 补漏、回填、定时任务失败后重跑 |
| 交互 | 日期选择器 +「开始更新」按钮；运行中按钮禁用 |
| 状态展示 | 任务队列列表：任务 ID、目标日期、触发方式（手动/定时）、状态、进度摘要、开始/结束时间 |
| 报错展示 | 同 4.7.1；失败任务可 **一键重试** |
| 并发 | 同一时刻 **仅允许 1 个采集任务** 运行（避免 API 限频与 DB 锁冲突） |

**数据拉取口径（指定日期）**

- 买卖：`history/transaction/{code}?st=YYYYMMDD&et=YYYYMMDD`（或等价单日参数）。
- 成交：优先历史日 K / 当日 ssjy 快照（实现见方案文档）；若 API 无法取历史成交，页面标注该日 `turnover` 缺失。
- ETF：若仅支持当日 `fd/real/time`，历史 ETF 仅能补 **已有 cron 积累** 的日期；手动补数对 ETF 显示能力边界提示。

#### 4.7.3 按日下载全量数据

| 项 | 说明 |
|----|------|
| 功能 | 选择某一已有数据的 `trade_date`，下载 **该日全部数据** 为文件包 |
| 格式 | **ZIP**，内含 UTF-8 **CSV** + `README.txt` |
| 文件清单 | `market_daily.csv`（1 行）、`sector_daily.csv`、`stock_daily.csv`、`etf_daily.csv`、`meta.json`（行数、生成时间、trade_date） |
| 交互 | 日期下拉（仅列出已有数据的日期）+「下载」按钮 |
| 空数据 | 该日无记录时按钮禁用并提示 |

#### 4.7.4 已有数据日期日历

| 项 | 说明 |
|----|------|
| 功能 | 展示系统中 **已落库** 的日期列表及完整度，便于掌握数据情况 |
| 展示形式 | **日历视图** + **列表视图**（可切换） |
| 每个日期显示 | `trade_date`；是否有 market / sector / stock / etf；各行数；最近更新时间 |
| 完整度标识 | ✅ 完整（四表均有且 stock_count > 0）／⚠ 部分（缺 ETF 或买卖）／❌ 失败任务残留 |
| 交互 | 点击某日 → 快捷 **下载** 或 **重新采集**（跳转 4.7.2 并预填日期） |

## 5. 数据需求（Phase 2）

看板统一依赖 **近 5 交易日** 历史，须先建设历史层。

### 5.1 最少历史深度

### 5.2 逻辑数据模型

#### market_daily（1 行 / 日）

| `active_sell` | 全 A 主卖 |

#### sector_daily（~31 行 / 日）

| 字段 | 说明 |
|------|------|

| `sector_name` | |
| `turnover`, `turnover_pct` | 成交额及占 A 股比 |
| `active_buy`, `active_sell`, `net_active` | |
| `stock_count` | |

#### stock_daily（~5200 行 / 日）

#### etf_daily（~1480 行 / 日）**新增**

| 字段 | 说明 |

| `etf_name` | |

#### fetch_jobs（采集任务日志）**新增**

| 字段 | 说明 |

#### app_settings（应用配置）**新增**

| 键 | 说明 | 默认 |
|----|------|------|
| `schedule_enabled` | 是否启用定时 | `true` |
| `schedule_time` | 每日执行时刻 HH:MM | `21:35` |
| `schedule_timezone` | 时区 | `Asia/Shanghai` |
| `schedule_enabled` | 是否启用定时 | `true` |
| `schedule_run_mode` | 执行日类型 | `trading_day`（默认）或 `calendar_day` |

#### trading_calendar（交易日历，本地缓存）**新增**

| 字段 | 说明 |
|------|------|
| `trade_date` | 主键，YYYY-MM-DD |
| `is_trading` | 1=交易日，0=休市 |
| `source` | `biying_kline` / `manual` |
| `updated_at` | 同步时间 |

**交易日判定来源**（见 [BIYING_API.md §交易日历](./BIYING_API.md)）：

- 必盈 **无独立「交易日历 list」接口**（`hslt/jyrl` 等路径实测 404）。
- 通过 **指数/个股日 K** `hsstock/history/{code}.SZ/d/n/{licence}?lt=N` 返回的 `t` 字段提取交易日列表，写入本地 `trading_calendar` 表。
- 调度器、看板「近 5 日」、管理页数据日历均优先读本地表；每周或每次服务启动时刷新未来 30 日 + 过去 1 年。

### 5.3 采集策略

| 场景 | 做法 |
|------|------|
| **每日定时** | 管理页配置（默认 **21:35**、`trading_day`）→ 调度器判断当日是否执行 → `fetch_by_daily.py` |
| **手动指定日** | 管理页触发 `fetch_by_date.py --date YYYY-MM-DD` |
| **首次 5 日回填** | `backfill_history.py --days 5` |
| **自然积累** | 连续采集 **5 个交易日** 后看板数据完整 |

### 5.4 API 频率（每日增量，与 Phase 1 相同）

| 数据 | 请求量级 |
|------|----------|
| A 股成交 | ~261 批 ssjy_more |
| A 股买卖 | ~5208 次 transaction |
| ETF 成交 | ~1480 次 fd/real/time |

## 10. 验收标准（Phase 3）

1. **页面 1**：三张柱图均为近 5 日，字段与 `market_daily` 一致，单位正确。
2. **页面 2**：5 个日组 × 2 列，占比排序默认降序，点击跳转页面 3 有效。
3. **页面 3**：每板块 3 图可折叠，「查看个股」进入页面 4。
4. **页面 4**：仅展示该板块成份股，每股 3 图可折叠。
5. **页面 5**：ETF 表格 5 日组成交 + 占 A 股比，排序与搜索可用，1480 行不卡顿。
6. **页面 6**：每 ETF 1 图可折叠，懒加载；与页面 5 排序/定位一致。
7. 历史不足 5 日时，各页展示实际天数并提示。

### 验收标准（Phase 3b 管理页）

1. 默认定时 **21:35**、执行日类型 **交易日**，修改后持久化并按新规则触发。
2. 手动选择日期触发采集，任务列表实时展示状态；失败可见错误信息与日志。
3. 已有数据的日期可下载 ZIP，内含 market / sector / stock / etf 四表 CSV。
4. 日历/列表正确展示已落库日期与各表行数；部分数据日期标 ⚠。
5. 并发提交两个手动任务时，第二个排队或拒绝并提示。
